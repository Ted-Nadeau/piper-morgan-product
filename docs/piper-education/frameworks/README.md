# Frameworks

**Purpose**: Architectural and development frameworks that provide structure and guidance for system design and implementation

## Organization

### **Established Frameworks** ([./established/](./established/))

Received wisdom and best practices that we consciously adopted and applied.

**Key Characteristics**:

- Well-documented and widely adopted
- Include citations and references to original sources
- Consciously chosen and implemented
- Examples: Domain-Driven Design, Test-First Development, ADR Pattern

### **Emergent Frameworks** ([./emergent/](./emergent/))

Frameworks that emerged organically through our development process.

**Key Characteristics**:

- Discovered through problem-solving and iteration
- Include evolution history and problem-solution mapping
- Emerged from real-world challenges
- Examples: Session Log Framework, Human-AI Collaboration Framework

#### **Critical Emergent Frameworks**

**Session Log Framework** ([./emergent/session-log-framework.md](./emergent/session-log-framework.md))

- **Pattern Strength**: 16/16 (Maximum Strength)
- **Purpose**: Maintain context, institutional memory, and effective handoffs
- **Key Features**: Structured session logs, handoff protocols, archive system
- **Real Examples**: Foundation Sprint, PM-012 GitHub Integration, PM-021 Error Handling

## Framework Categories

### **Architectural Frameworks**

- System design and structure patterns
- Component organization and interaction
- Scalability and performance patterns

### **Development Frameworks**

- Development workflow and process patterns
- Code organization and structure
- Quality assurance and testing patterns

### **Integration Frameworks**

- System integration and communication patterns
- API design and implementation
- External service integration patterns

## Usage Guidelines

1. **Start with Established**: Begin with established frameworks for proven approaches
2. **Enhance with Emergent**: Use emergent frameworks to address specific challenges
3. **Combine Approaches**: Mix established and emergent frameworks as appropriate
4. **Document Evolution**: Track how frameworks evolve and improve over time
5. **Share Discoveries**: Contribute emergent frameworks back to the community
