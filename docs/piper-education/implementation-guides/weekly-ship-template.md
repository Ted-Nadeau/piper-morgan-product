## 📋 Template Notes

### Adaptation Guidelines:

- **Engineering Focus**: Emphasize technical achievements, performance metrics, architecture decisions
- **Learning Integration**: Dedicated section for patterns that could help other teams
- **Resource Transparency**: Time allocation instead of budget (more relevant for internal team)
- **Public-Ready**: No proprietary information, technical details linked rather than embedded

### Weekly Preparation Process:

1. **Thursday Review**: With dev lead, chief architect, chief of staff, comms lead
2. **Content Audit**: Review session logs, blog drafts, sprint progress
3. **Pattern Identification**: Use AI analysis of logs/content for learning section
4. **Draft & Edit**: Friday morning writing, afternoon distribution
5. **Distribution**: Notion page + Slack #piper-morgan + optional broader sharing

### AI-Assisted Content Analysis:

- **RAG on Session Archive**: Query corpus for technical patterns, decision rationales
- **Learning Detection**: LLM agent scans logs for "interesting insights" based on initial taxonomy
- **Trend Analysis**: Identify recurring themes across multiple sessions
- **Impact Measurement**: Track which shared learnings get adopted by other teams
