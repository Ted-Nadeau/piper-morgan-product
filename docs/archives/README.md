## Archive Organization

### Session Logs (`session-logs/`)
**Chronological organization by year and month**
```
session-logs/
├── 2025/
│   ├── 08/
│   │   ├── index.md              # Monthly summary and navigation
│   │   └── 2025-08-*.md         # Daily session logs
│   └── 09/
│       ├── index.md
│       └── 2025-09-*.md
├── yearly-index.md               # High-level navigation by year
└── omnibus-logs/                 # Weekly/monthly consolidations
```

### Historical Decisions (`decisions/`)
- **Completed ADRs** moved from active architecture
- **Historical planning decisions** with context preservation
- **Methodology evolution** documentation
- **Decision rationale** and outcome tracking

### Generated Artifacts (`artifacts/`)
- **Session deliverables** co-located with related logs
- **Generated reports** and analysis outputs
- **Historical exports** and documentation snapshots
- **Tool outputs** and automated generation results

## Archaeological Research Features

### Session-Based Artifact Co-location
- Session logs grouped with related deliverables
- Cross-references between related sessions
- Blog draft identification and tracking
- Agent role and delivery tracking

### Navigation Aids
- **Monthly indices** for efficient browsing
- **Topic-based cross-references** across sessions
- **Agent-based filtering** by role (Chief Architect, Lead Developer, etc.)
- **Deliverable tracking** by type and impact

### Research Methodology
1. **Chronological access** - Browse by date for development timeline
2. **Topic clustering** - Find related work across time periods
3. **Agent perspective** - Understand role-specific contributions
4. **Impact tracking** - Trace decisions from session to implementation

## Preservation Standards

### Session Log Integrity
- **Never modify** historical session logs
- **Preserve original context** and cross-references
- **Maintain metadata** including timestamps and agent roles
- **Archive co-location** of related artifacts

### Decision Documentation
- **Complete context** preservation for historical decisions
- **Outcome tracking** for implemented changes
- **Rationale preservation** for future reference
- **Link maintenance** to related sessions and artifacts

### Artifact Management
- **Source preservation** for generated content
- **Version tracking** for iterative deliverables
- **Cross-reference maintenance** between artifacts and sessions
- **Archaeological tagging** for research categorization
