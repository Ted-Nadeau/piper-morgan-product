# Documentation Sync Checklist

## Issue Closure Process (After Every Work Session)

### Step 1: GitHub Issue Status Update

- [ ] **Close GitHub Issue**: Mark as completed with completion evidence
- [ ] **Add Completion Comment**: Include key achievements and metrics
- [ ] **Link Related Issues**: Reference dependencies and follow-up items
- [ ] **Update Labels**: Mark priority, component, and status appropriately

### Step 2: CSV File Synchronization

- [ ] **Update pm-issues-status.csv**: Change status from OPEN to CLOSED
- [ ] **Verify Issue Number**: Ensure PM number matches GitHub issue
- [ ] **Add Completion Date**: Record when issue was completed
- [ ] **Check for New Issues**: Add any new issues created during session

### Step 3: Document Updates

- [ ] **Update Backlog**: Move completed items to appropriate sections
- [ ] **Update Roadmap**: Reflect completed achievements in strategic direction
- [ ] **Update Completed.md**: Add new achievements to archive
- [ ] **Remove from Active**: Ensure completed items don't appear in active work

### Step 4: Session Log Maintenance

- [ ] **Update Session Log**: Document what was accomplished
- [ ] **Record Metrics**: Include performance data and success criteria
- [ ] **Note Learnings**: Document insights and methodology improvements
- [ ] **Plan Next Steps**: Identify follow-up actions and dependencies

## Weekly Documentation Audit

### Monday: GitHub Issues Review

- [ ] **Check Open Issues**: Review all open GitHub issues for accuracy
- [ ] **Update Priorities**: Adjust priority levels based on current sprint
- [ ] **Verify Dependencies**: Ensure dependency relationships are correct
- [ ] **Clean Up Labels**: Remove outdated or incorrect labels

### Wednesday: CSV Synchronization

- [ ] **Sync with GitHub**: Update CSV to match GitHub reality
- [ ] **Check for Drift**: Identify any discrepancies between sources
- [ ] **Update Statuses**: Reflect current issue status accurately
- [ ] **Add Missing Issues**: Include any issues not in CSV

### Friday: Document Review

- [ ] **Review Backlog**: Ensure active work is accurately reflected
- [ ] **Update Roadmap**: Adjust strategic direction based on progress
- [ ] **Archive Completed**: Move completed items to appropriate archives
- [ ] **Plan Next Week**: Identify priorities for upcoming sprint

## Monthly Documentation Cleanup

### Archive Management

- [ ] **Review Session Logs**: Archive month N-2 sessions
- [ ] **Update Retention Policy**: Ensure proper active vs archived separation
- [ ] **Clean Up Duplicates**: Remove any duplicate or outdated content
- [ ] **Verify References**: Check for broken links or references

### Strategic Review

- [ ] **Update Strategic Roadmap**: Reflect current market position
- [ ] **Review Success Metrics**: Update based on recent achievements
- [ ] **Plan Next Quarter**: Identify strategic priorities and goals
- [ ] **Update Competitive Analysis**: Reflect current market position

## Quality Assurance

### Accuracy Checks

- [ ] **Cross-Reference Verification**: Ensure all sources match
- [ ] **Status Consistency**: Verify GitHub, CSV, and documents align
- [ ] **Reference Integrity**: Check for broken links or missing files
- [ ] **Content Freshness**: Ensure information is current and relevant

### Completeness Validation

- [ ] **Issue Coverage**: All GitHub issues represented in documentation
- [ ] **Achievement Documentation**: All completed work properly archived
- [ ] **Strategic Alignment**: Roadmap reflects current strategic direction
- [ ] **Process Documentation**: All processes and procedures documented
