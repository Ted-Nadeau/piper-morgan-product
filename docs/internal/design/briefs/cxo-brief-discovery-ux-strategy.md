# Chief Experience Officer Brief: Discovery-First UX Strategy
**Date**: December 27, 2025
**From**: Chief Architect
**Re**: Conversational Glue and MUX-INTERACT Priorities

## Executive Summary

Pattern-045 (Green Tests, Red User) demonstrates our features work but users can't find them. The conversational glue design brief addresses this systematically. MUX-INTERACT must be discovery-first, not feature-first.

## Three-Level UX Evolution

1. **Level 1 - Explicit** (Now): Setup wizard, preference panels, manual configuration
2. **Level 2 - Conversational** (Beta): "Set up my calendar integration" natural language
3. **Level 3 - Proactive** (Future): "I notice you mention sprints but have no calendar..."

## Pattern Insights

- **Pattern-045**: Technical success ≠ user success
- **Discovery Friction**: 19 working queries, 0% user discovery rate
- **Conversational Paradigm**: Command-response → colleague dialogue
- **MUX Opportunity**: Consciousness model enables natural discovery

## Questions for Your Expertise

1. **Discovery Patterns**: What specific UI patterns have you seen successfully bridge the gap between "feature exists" and "user finds feature"? Should we consider:
   - Progressive disclosure menus?
   - Contextual command palette?
   - Learning mode vs expert mode?

2. **Conversational Design Philosophy**: Our current paradigm is command-response. What interaction principles should guide the shift to conversational colleague? How do we avoid the "uncanny valley" of pseudo-human interaction?

3. **Onboarding vs Discovery**: Is discovery failure actually an onboarding failure? Should we redesign FTUX to teach discovery patterns rather than features?

4. **Capability Revelation**: How should Piper reveal new capabilities?
   - Proactive suggestions risk being annoying
   - Passive waiting risks never being discovered
   - What's the sweet spot?

5. **Mental Model Formation**: Users need to form an accurate mental model of Piper's capabilities. What UX patterns help users understand not just "what" but "how much" an AI assistant can do?

## Your Input Needed On

## Specific Design Challenges

1. **The Blank Prompt Problem**: User opens Piper, sees empty chat, doesn't know what to type
2. **The Capability Ceiling**: User assumes Piper can only do what they've already seen
3. **The Discovery Gradient**: Moving users from simple to complex capabilities
4. **The Context Switch**: Slack vs Web vs CLI - maintaining discovery across channels
