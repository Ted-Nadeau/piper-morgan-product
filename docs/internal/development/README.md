## Usage Guidelines

### For Active Development
1. **Start work** - Place initial files in `active/in-progress/`
2. **Follow methodology** - Reference appropriate methodology from `methodology-core/`
3. **Use tools** - Leverage setup guides and workflows from `tools/`
4. **Coordinate** - Use handoff patterns for agent collaboration
5. **Complete work** - Move finished items to `ready-for-integration/`

### For Code Reviews
1. **Check pending-review** - Files awaiting feedback
2. **Apply methodology** - Use quality standards from methodology core
3. **Coordinate feedback** - Use handoff patterns for communication
4. **Update status** - Move items based on review outcomes

### For Planning
1. **Reference methodology** - Use systematic approaches from core
2. **Check current planning** - Review active planning documents
3. **Coordinate resources** - Use planning templates and frameworks
4. **Track progress** - Update status in appropriate active directories
