# Suggestions (Pre-Issue)

**Purpose**: Capture ideas, change requests, and improvement suggestions **before** they become formal GitHub issues. Use this when something is worth recording but not yet triaged, scoped, or committed.

**Relationship to issues**:
- **Suggestions** = early-stage: "we might want…", "consider…", "could we…"
- **GitHub issues** = triaged: clear acceptance criteria, assigned, or in backlog

When a suggestion is ready to be worked on, promote it to a GitHub issue (and optionally note the issue number here or move the line to "Promoted" section).

## Format

Each suggestion entry can include:
- **Title** (one line)
- **Context** (optional: why, where it came from)
- **Status**: `open` | `drafted` (artifact exists; ready for review/promotion) | `promoted` (issue #NNN) | `deferred` | `rejected`

No strict template—keep it lightweight so it's easy to add and later promote.
