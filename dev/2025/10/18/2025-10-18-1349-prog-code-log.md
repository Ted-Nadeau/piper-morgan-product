```python
# Updated constructor
def __init__(self, config_service: Optional[GitHubConfigService] = None):
    self.mcp_adapter = None  # NEW: MCP adapter instance
    self.spatial_github = None  # Existing: Spatial fallback

    # Feature flag control
    self.use_mcp = self._get_boolean_flag("USE_MCP_GITHUB", True)

    # Initialize integrations (MCP preferred, spatial fallback)
    self._initialize_integrations()

# New initialization logic
def _initialize_integrations(self):
    # Try MCP adapter first (if enabled)
    if self.use_mcp:
        try:
            self.mcp_adapter = GitHubMCPSpatialAdapter()
            logger.info("GitHubMCPSpatialAdapter initialized successfully")
        except Exception as e:
            logger.warning(f"MCP init failed: {e}, falling back")
            self.mcp_adapter = None

    # Initialize spatial as fallback
    try:
        self.spatial_github = GitHubSpatialIntelligence()
        logger.info("GitHubSpatialIntelligence initialized successfully")
    except Exception as e:
        if not self.mcp_adapter:
            raise RuntimeError("No GitHub integration available") from e
        logger.warning(f"Spatial failed, using MCP only: {e}")

# Updated delegation
def _get_integration(self, operation: str):
    # Prefer MCP when available
    if self.mcp_adapter:
        return self.mcp_adapter
    if self.spatial_github:
        return self.spatial_github
    raise RuntimeError(f"No GitHub integration available for {operation}")

# Added boolean flag parser
def _get_boolean_flag(self, flag_name: str, default: bool = False) -> bool:
    """Parse boolean env vars: true/false/1/0/yes/no/on/off"""
    try:
        value = os.getenv(flag_name, str(default)).lower().strip()
        return value in ("true", "1", "yes", "on", "enabled")
    except Exception:
        return default
```

## Architectural Insights

### Pattern Established: Tool-Based MCP Integration

Both Calendar and GitHub now follow this pattern:

```
┌─────────────────────────────────────────────────────┐
│         Integration Router                          │
│  (CalendarIntegrationRouter / GitHubIntegrationRouter)│
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────────────┐      ┌──────────────────┐   │
│  │  MCP Adapter    │      │  Spatial         │   │
│  │  (PRIMARY)      │ ───▶ │  Intelligence    │   │
│  │                 │      │  (FALLBACK)      │   │
│  └─────────────────┘      └──────────────────┘   │
│         ▲                                          │
│         │                                          │
│    Feature Flag                                    │
│  (USE_MCP_*)                                       │
└─────────────────────────────────────────────────────┘
         ▲
         │
    ┌─────────────────┐
    │  Config Service │
    │  (Injected)     │
    └─────────────────┘
```

**Key Components**:
1. **MCP Adapter** (Primary) - Tool-based MCP protocol integration
2. **Spatial Intelligence** (Fallback) - Direct API with spatial enhancement
3. **Feature Flag Control** - Runtime switching (USE_MCP_CALENDAR, USE_MCP_GITHUB)
4. **Service Injection** - Config service for dependency injection
5. **Graceful Degradation** - Falls back if MCP unavailable

### Configuration Priority Pattern

Both integrations implement three-layer configuration:

```
1. Environment Variables (HIGHEST)
   ↓ overrides
2. PIPER.user.md (MIDDLE)
   ↓ overrides
3. Hardcoded Defaults (LOWEST)
```

**Implementation**:
```python
def _load_config(self):
    # Load from PIPER.user.md first (base layer)
    user_config = self._load_from_user_config()

    # Environment variables override user config
    return Config(
        setting=os.getenv(
            "ENV_VAR",
            user_config.get("setting", "default_value")
        )
    )
```

**Benefits**:
- User customization without repository commits
- Environment-based overrides for deployment
- Sensible defaults for quick start
- Gitignored user configs (credentials stay local)

Three-layer system (env > user > defaults) provides:
- Developer flexibility (env vars for testing)
- User customization (PIPER.user.md for personal settings)
- Sensible defaults (quick start without config)
