**Found**:
- `IntentService.process_intent()` in `services/intent/intent_service.py`
- Routes intents based on category, NOT through canonical_handlers for everything

**Key Discovery**: Routing is category-based, not action-based:
```python
# services/intent/intent_service.py:84
async def process_intent(message, session_id):
    intent = await classifier.classify(message)

    if intent.category == "CONVERSATION":
        return await _handle_conversation_intent(intent, session_id)

    if intent.category == "QUERY":
        return await _handle_query_intent(intent, workflow, session_id)

    # EXECUTION/ANALYSIS → placeholder
    return await _handle_generic_intent(intent)
```

### Step 2: Trace End-to-End Flow ✅

**API Endpoint**: `POST /api/v1/intent` → `IntentService.process_intent()`

**Flow**:
1. `IntentClassifier.classify()` → Intent object
2. `IntentService.process_intent()` → routes by category
3. Different handlers for different categories:
   - CONVERSATION → ConversationHandler
   - QUERY (specific) → StandupOrchestrationService, ProjectsQuery
   - QUERY (general) → OrchestrationEngine.handle_query_intent() → QueryRouter
   - TEMPORAL/STATUS/PRIORITY/IDENTITY/GUIDANCE → CanonicalHandlers
   - EXECUTION/ANALYSIS → Placeholder message (Phase 3C)
