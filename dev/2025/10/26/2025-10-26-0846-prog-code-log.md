## Part 5: Knowledge Graph + Preferences + Patterns

### How They Connect

1. **Pattern Learning** → Detects user behavior patterns (e.g., "prefers morning meetings")
2. **Preference Persistence** → Stores learned pattern as explicit preference
3. **Knowledge Graph** → Enriches intent classification with user's known preferences
4. **Intent Classification** → Uses graph context (including preferences) to improve classification

### Concrete Example: Morning Meeting Preference

```
1. User says: "I prefer morning meetings because I have more energy"

2. QueryLearningLoop.learn_pattern():
   - Type: USER_PREFERENCE_PATTERN
   - Data: { preference_key: "meeting_time", preference_value: "morning" }
   - Confidence: 0.85

3. LearnedPattern.apply_pattern() converts to explicit preference:
   - UserPreferenceManager.apply_preference_pattern(pattern, user_id="xian")
   - Sets: preferences["meeting_time"] = "morning" at user scope

4. Knowledge graph stores relationship:
   - Node: "morning_meeting"
   - Node: "high_energy"
   - Edge: ENABLES (morning_meeting → high_energy, confidence: 0.9)

5. Next time user says: "When should we schedule the architecture review?"
   - IntentClassifier.classify():
     - Gets graph context via get_relevant_context()
     - Finds reasoning chain: morning_preference ENABLES high_energy
     - Extracts hints: ["morning", "enables", "high_energy"]
     - These hints inform classification
     - Response can consider the user's morning preference
```

## Part 6: Current Wiring Status

### ✅ What's Wired

1. **KnowledgeGraphService** → **IntentClassifier**
   - Graph context passed via classify(context)
   - _get_graph_context() calls service.get_relevant_context()
   - Hints extracted and available to classification

2. **QueryLearningLoop** → **UserPreferenceManager**
   - apply_pattern() explicitly calls apply_preference_pattern()
   - USER_PREFERENCE_PATTERN type has dedicated handler
   - Confidence threshold enforced (>= 0.7)

3. **QueryLearningLoop** → **OrchestrationEngine**
   - learning_loop initialized in engine.__init__()
   - Patterns learned during query handling
   - Pattern metadata includes feature source

### ⚠️ What's NOT Fully Wired

1. **Graph Context Not Used in Default Classify**
   - classify() without user_id doesn't get graph context
   - Would need to pass user_id in context parameter

2. **No Automatic Pattern→Preference Application**
   - Patterns are learned but not automatically applied as preferences
   - Requires explicit apply_pattern() call
   - Would need middleware to auto-apply high-confidence patterns

3. **No Closed-Loop Feedback Integration**
   - Feedback system exists but not integrated with classification
   - Could improve patterns based on user feedback
