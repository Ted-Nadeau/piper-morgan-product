## LESSONS LEARNED

### What Worked Exceptionally Well

1. **TDD Approach**: Writing tests first caught issues early and ensured quality
2. **Modern Pattern**: Intent/IntentProcessingResult pattern is clean and consistent
3. **Helper Methods**: Breaking complex handlers into helpers improved readability
4. **Serena Tools**: Symbolic editing made large-scale changes precise and efficient
5. **Phase Structure**: 6-part structure (Study → Scope → Tests → Implement → Run → Evidence) was repeatable

### Challenges Overcome

1. **Service Access**: Learned local import pattern for services (avoid circular deps)
2. **Output Structures**: Tests drove structured, consistent output formats
3. **Validation**: Comprehensive validation prevented runtime errors
4. **Edge Cases**: TDD caught edge cases like empty data, missing parameters
5. **Pattern Variations**: Each category required unique approaches (analysis ≠ synthesis ≠ strategy)

### Best Practices Established

1. Always validate inputs comprehensively
2. Provide actionable error messages with clarification types
3. Structure output consistently across handlers
4. Generate recommendations, not just data
5. Handle edge cases gracefully (no data, empty inputs)
6. Log at appropriate levels (debug, info, warning, error)
7. Use type hints for clarity
8. Write comprehensive docstrings
9. Test all code paths
10. Collect evidence after each phase
