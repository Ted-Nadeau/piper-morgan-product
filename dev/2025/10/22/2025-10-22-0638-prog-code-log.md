### Step 1: Added rotate_user_key Method
**File**: `services/security/user_api_key_service.py` (lines 301-391, now 396 lines total)

**Method Signature**:
```python
async def rotate_user_key(
    self,
    session: AsyncSession,
    user_id: str,
    provider: str,
    new_api_key: str,
    validate: bool = True
) -> UserAPIKey:
```

**Zero-Downtime Strategy**:
1. Get existing key record from database
2. Raise ValueError if no existing key found
3. Store old key reference for rollback capability
4. Validate new key (optional but recommended)
5. Store new key in keychain (overwrites old key)
6. Update database with rotation metadata:
   - `previous_key_reference` = old reference
   - `rotated_at` = current timestamp
   - `is_validated` = validation result
7. Commit transaction (atomic operation)

**Rotation Metadata Tracked**:
- `previous_key_reference`: Old keychain reference (for rollback)
- `rotated_at`: Timestamp of rotation
- `is_validated`: Whether new key was validated

**Error Handling**:
- Raises `ValueError` if no existing key found
- Raises `ValueError` if validation fails
- Raises `ValueError` if keychain storage fails
