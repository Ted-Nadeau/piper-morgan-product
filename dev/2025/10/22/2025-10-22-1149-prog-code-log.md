### Smart Resume Implementation (12:32 PM - 12:35 PM)

**Feature**: Detect incomplete setup and offer to resume.

**Logic**:
1. Before prompting for username, check for users with no API keys
2. If found: "Found incomplete setup for: {username}" with email
3. Ask: "Resume this setup? (y/n)"
4. If yes: Skip username/email prompts, go directly to API key collection
5. If no: Continue with new username prompt (existing account remains)

**Implementation Details**:

Created `check_for_incomplete_setup()` function:
```python
async def check_for_incomplete_setup() -> Optional[User]:
    """
    Check if there's a user with no API keys (incomplete setup).

    Returns:
        User object if incomplete setup found, None otherwise
    """
    async with AsyncSessionFactory.session_scope() as session:
        # Find users with no API keys (LEFT JOIN where no match)
        result = await session.execute(
            select(User)
            .outerjoin(UserAPIKey)
            .where(UserAPIKey.id.is_(None))
            .order_by(User.created_at.desc())
            .limit(1)
        )
        user = result.scalar_one_or_none()
        return user
```
