### Input Parsing Logic

```python
def parse_choice(input_str: str, options: List[str]) -> str:
    """Parse user input (1-N) to option value"""
    try:
        choice = int(input_str.strip())
        if 1 <= choice <= len(options):
            return options[choice - 1]
    except (ValueError, IndexError):
        pass

    # Default to middle option if invalid input
    return options[len(options) // 2]
```

**Graceful Defaults**:

- Invalid input → middle option (index `len(options) // 2`)
- "abc" → "balanced" (for 3 options)
- "0" or "4" → "balanced" (for 3 options)
- Empty input → "balanced" (for 3 options)

### Error Handling Strategy

**Database Errors**:

- User not found in `alpha_users` table → clear error message
- Connection failures → graceful fallback with retry suggestion
- SQL errors → logged but user-friendly message shown

**User Input Errors**:

- Invalid choices → default to middle option (no error shown)
- Keyboard interrupt → clean exit with message
- Empty input → treated as middle option

**System Errors**:

- Import failures → clear error messages
- Async operation failures → proper exception handling
