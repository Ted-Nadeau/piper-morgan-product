## Technical Implementation

### Provider Format Rules

**OpenAI**: `sk-[A-Za-z0-9]{48,}` (51+ chars)
**Anthropic**: `sk-ant-[A-Za-z0-9\-_]{100,}` (108+ chars)
**GitHub**: `gh[pous]_[A-Za-z0-9]{36,}` (40+ chars)
**Perplexity**: `pplx-[A-Za-z0-9\-_]{35,}` (40+ chars)
**Gemini**: `AIza[A-Za-z0-9\-_]{35,}` (39+ chars)

### Strength Scoring Algorithm

```python
# Weighted scoring (0.0 to 1.0)
overall_score = (
    length_score * 0.2 +      # Length adequacy
    entropy_score * 0.3 +     # Randomness measure
    diversity_score * 0.2 +   # Character type variety
    pattern_score * 0.3       # Absence of weak patterns
)

# Security levels
strong: >= 0.8
medium: >= 0.6
weak: < 0.6
```

### Leak Detection Layers

1. **Known Test Keys**: Database of common examples/placeholders
2. **Weak Patterns**: Detection of "test", "demo", "example" patterns
3. **Fake Key Detection**: Sequential/repeated character analysis
4. **Future**: HIBP integration for public leak databases
