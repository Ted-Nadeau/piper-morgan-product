### Health Endpoint Timezone Issue

**Problem**: `/api/v1/standup/health` returns local time without timezone marker

**Current behavior**:
```json
{
  "timestamp": "2025-10-20T07:06:43.159681"  // Local time, no timezone
}
```

**Expected behavior**:
```json
{
  "timestamp": "2025-10-20T14:06:43.159681Z"  // UTC time with 'Z' marker
}
```

**Impact**:
- Causes confusion when comparing timestamps
- ISO 8601 recommends UTC with explicit timezone
- Integration test had to work around this

**Recommendation**: File separate issue to fix health endpoint timestamp format
