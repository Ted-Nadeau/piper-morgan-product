## Session Capture with `tee` (Terminal Workflow Enhancement)

**Current State**: PM may not be capturing terminal Claude Code sessions automatically

**Proposed Enhancement**: Use `tee` to capture ALL terminal output (both Claude's responses and command results) to session log files.

### How It Works

```bash
# Terminal Claude Code with automatic session capture
claude-code | tee dev/active/2025-11-19-0647-prog-code-log.md

# Everything you see gets written to the log file:
# - My responses
# - Bash command output
# - Pytest results
# - Git operations
# - Error messages
```

### Benefits

1. **Zero-Effort Logging**: Don't manually copy/paste to session logs
2. **Complete Audit Trail**: Every command, every output, timestamped
3. **Evidence Preservation**: Test results automatically captured for GitHub issues
4. **Reproducibility**: Exact terminal state preserved
5. **Chief of Staff Review**: Complete session history without reconstruction

### Example Workflow

```bash
# Start terminal Claude Code session with logging
cd /Users/xian/Development/piper-morgan
claude-code | tee dev/active/$(date +%Y-%m-%d-%H%M)-prog-code-log.md

# Now everything is captured:
# - Claude: "I'll run the tests..."
# - [bash command output]
# - Claude: "Here's what I found..."
# All goes to both stdout AND the log file
```

### Comparison to Manual Logging

**Manual (current?)**:
- Claude creates session log with Write tool
- PM copies interesting bits
- Evidence screenshots/pastes added manually
- Incomplete record of session

**With tee (proposed)**:
- Complete verbatim session transcript
- All command output preserved
- No manual effort
- Easy to share with chief of staff
