**2. Scope Discipline**

- Global and per-pattern controls

**4. Infrastructure Reuse**
- Sprint A5 work provides foundation
- Pattern Sweep continues for codebase learning

### Architectural Suggestions

**1. Pattern Storage Strategy**
Consider partitioning patterns by type early:
```python
# Separate tables for different pattern lifecycles
user_patterns      # Per-user, high churn
team_patterns      # Future: shared, medium churn
system_patterns    # Global PM best practices, low churn
```

**2. Confidence Decay**
Add time-based confidence decay to handle changing behaviors:
```python
def get_effective_confidence(pattern):
    days_since_use = (now - pattern.last_used_at).days
    decay_factor = 0.95 ** days_since_use  # 5% decay per day
    return pattern.confidence * decay_factor
```

**3. Pattern Categories**
Group patterns for better UX:
- **Workflow Patterns**: Multi-step sequences
- **Timing Patterns**: "After standup", "End of day"
- **Preference Patterns**: Tool settings, formats
- **Command Patterns**: Frequently used commands

**4. Privacy First**
Explicitly state in code and docs:
- Patterns stored locally (user's data only)
- No cross-user analysis in Basic Auto
- Clear data deletion on account removal
