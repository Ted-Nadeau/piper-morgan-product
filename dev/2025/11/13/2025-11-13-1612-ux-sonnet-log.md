**Approach Taken**: "Thoughtful Colleague" pattern

## Decisions Made

### Design Decisions (Pattern Suggestions - Web)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Display Pattern** | Progressive disclosure | Non-intrusive, discoverable, scalable |
| **Initial State** | Collapsed with badge | Reduces noise, user-initiated |
| **Suggestion Count** | Top 3 | Focused without overwhelming |
| **Confidence Threshold** | 0.7 (70%) | Balance between quality and quantity |
| **Feedback Actions** | Accept/Reject/Dismiss | Clear options, handles false positives |
| **Qualitative Feedback** | Optional text input | Values explicit feedback over inference |
| **On Accept** | Record only (no execution) | Clean phase separation, safer |
| **Visual Style** | Teal badge, expandable cards | Aligns with Piper brand, professional |
| **Mobile Pattern** | Stacked buttons, full width | Touch-friendly, readable |
| **Accessibility** | WCAG 2.1 AA compliant | Keyboard nav, screen reader support |

### Process Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Testing** | Manual (like Phase 2) | Quality standard, catches UI issues early |
| **Documentation** | Comprehensive proposal | Enables implementation, future reference |
| **Code Snippets** | Included in proposal | Accelerates development, shows feasibility |
| **Success Metrics** | Quantitative + qualitative | Validates design, informs iteration |
