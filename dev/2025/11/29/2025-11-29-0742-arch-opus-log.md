## Memo Review: Agent Coordination Patterns

### Formalized Prompt Queue (High Priority)

The Chief of Staff's proposal for a self-service job queue is architecturally sound and addresses our immediate needs better than full Agent Mail adoption. The structure:

```
/coordination/
  manifest.json
  /available/
  /claimed/
  /complete/
  /blocked/
```

**Architectural Assessment**: This pattern provides several advantages:
- **Decoupling**: Separates planning (prompt creation) from execution (agent work)
- **Scalability**: Multiple agents can work in parallel without coordination overhead
- **Auditability**: Git-tracked history of all work
- **Simplicity**: No complex messaging protocol needed

**Recommendation**: Implement immediately with these enhancements:
1. Add a `priority_score` field combining urgency and impact
2. Include `max_claim_duration` to prevent indefinite holds
3. Add `retry_count` for prompts that fail verification

### Advisor Mailbox for Ted (Medium Priority)

The recursive learning opportunity is elegant - Ted builds his own interface while learning the codebase. This pattern could extend to:
- Other advisors (Christina, etc.)
- External alpha testers providing structured feedback
- Eventually, a model for how Piper enables multi-stakeholder collaboration

**Architectural Note**: Keep the initial implementation file-based for simplicity, but design with future migration to database storage in mind.

## 8:30 AM - Models Audit Complete ✓

Created comprehensive audit document identifying critical gaps:

### Key Findings
- **No Moment concept** - Everything is task/item based
- **No consciousness** - Models are data holders, not actors
- **No Situation container** - Nothing groups moments meaningfully
- **No lifecycle transformation** - Death is deletion, not composting
- **Ownership model missing** - No Native/Federated/Synthetic

### Critical Insight
Morning Standup succeeds by imposing consciousness DESPITE the models, not through them. It adds "I notice" and "I'm concerned" at the application layer because the models can't express it.

### Remediation Priority
1. **Phase 1**: Create Moment model (16 hrs)
2. **Phase 2**: Add consciousness to Entities (24 hrs)
3. **Phase 3**: Implement Situation container (12 hrs)
4. **Phase 4**: Add ownership model (8 hrs)

Total effort: 60+ hours, but essential for consciousness preservation.
