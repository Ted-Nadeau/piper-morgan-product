## Agent Mail Research Discussion

### Research Conducted

Deep research into Jeffrey Emanuel's MCP Agent Mail server and Steve Yegge's "Agent Village" concept (Beads + Agent Mail). Key findings:

- Agent Mail provides: memorable identities, async messaging, file reservations
- Combined with shared memory, enables emergent self-organization ("no ego, quickly decide on leader")
- Jeffrey Emanuel ran 7 concurrent agents with 1,000+ messages
- Industry converging on MCP (tools) + A2A (agents) as complementary protocols

### Recommendations Developed

**For Methodology:**
- File reservation pattern: LOW priority (human coordination already handles this)
- Formalized Prompt Queue: HIGH priority (decouples planning from execution)
- Advisor Mailbox for Ted: MEDIUM-HIGH priority (enables async participation)

**For Product:**
- Adopt patterns, not full implementation
- Keep internal coordination ephemeral, not Git-persisted
- Don't expose to users; don't allow external agent access

### Prompt Queue Design

Developed concrete structure:
```
/coordination/
  manifest.json
  /available/   → prompts ready to claim
  /claimed/     → agent working
  /complete/    → done, verified
  /blocked/     → waiting on dependency
```

Enables: spin up agent → "check available work" → agent self-serves → spin up another → parallel execution without coordination overhead.

### Key Insight from PM

File reservation solves *upstream coordination* (preventing wasted parallel effort), not conflict resolution (git handles that). But current human coordination already provides this. The real value is the prompt queue pattern that decouples planning from execution.
