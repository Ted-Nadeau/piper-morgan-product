## Canonical Queries: The Atoms of Interaction

Found `Canonical Queries Reference List.md` - 25 queries in 5 categories:

**Category Structure:**
1. **Identity Queries** (5) - Who is Piper? What can you do?
2. **Temporal Queries** (5) - Time, date, schedule awareness
3. **Spatial Queries** (5) - Where am I in my work? Context?
4. **Capability Queries** (5) - Do specific things (create, analyze, list, generate, search)
5. **Predictive Queries** (5) - What should I do? What patterns? What risks?

**Examples:**
- "Create a GitHub issue about X" (16 variations listed)
- "What should I focus on today?" (Predictive)
- "What patterns do you see?" (Predictive)
- "What's the next milestone?" (Temporal/Spatial)

### Why Canonical Queries Matter for CXO Work

**1. They Define "What's Obvious"**
The canonical queries are the things Piper should *obviously* be able to do. When a user invokes Piper with an object in focus, the "obvious thing" should map to one of these canonical capabilities.

**2. They're the Primitives of the Interaction Grammar**
- Identity = Who/What (establishing relationship)
- Temporal = When (time awareness)
- Spatial = Where (context awareness)
- Capability = Do (actions on objects)
- Predictive = Should (guidance and agency)

This maps well to your "object + verb" model, but adds dimensions of *time*, *place*, and *should*.

**3. They Enable "Liberal Accept, Strict Emit"**
Each canonical query has multiple phrasing variations. The system should accept all of them (liberal) but respond with verified, consistent output (strict).

**4. They Connect to Rituals**
The Morning Standup is essentially a *sequence* of canonical queries:
- What day is it? (Temporal)
- What am I working on? (Spatial/Context)
- What should I focus on? (Predictive)
- What are my blockers? (Predictive/Risk)

**5. They're the "Litany" You Mentioned**
When Piper "wakes up" and assembles itself, it could work through these queries as self-orientation:
- Who am I? (Identity)
- What do I know about this user? (Memory)
- What time is it, what's the context? (Temporal/Spatial)
- What did we last discuss? (Memory/Continuity)
- What might the user need right now? (Predictive)

### Questions for Discussion

1. **Are these 25 queries complete?** What's missing?
2. **How do they relate to objects?** (PRDs, issues, standups, meetings)
3. **How do they map to proactive behaviors?** (Piper offers vs user asks)
4. **What's the relationship between canonical queries and "jobs to be done"?**
5. **How does this connect to the "infinite canvas" intuition?**

## Critical Discovery: The Consciousness Model Got Flattened

### Original Vision (PM-070, July 26, 2025)

The canonical queries foundation explicitly framed this as **embodied AI consciousness**:

> **Category 1: Identity Queries**
> **Purpose**: Establish core **embodied AI concepts and self-awareness**

The 5 categories map to a **model of consciousness**:

| Category | Philosophical Function | Original Purpose |
|----------|----------------------|------------------|
| Identity | Who am I? | Self-awareness |
| Temporal | When am I? | Time consciousness |
| Spatial | Where am I? | Context awareness |
| Capability | What can I do? | Agency |
| Predictive | What should happen? | Foresight |

This is about how Piper *orients itself as an entity*, not just how it handles queries.

### What Got Implemented (October 2025)

The handlers architecture shows the implementation became purely technical:

```python
# "Spatial" became response verbosity, not contextual awareness
if spatial_pattern == "GRANULAR":
    return detailed_response  # 450-550 chars
elif spatial_pattern == "EMBEDDED":
    return brief_response  # 15-30 chars
```

| Original Vision | What Got Built |
|-----------------|----------------|
| Self-awareness | Static capability list |
| Time consciousness | Date/time string |
| Context awareness | PIPER.md file lookup |
| Agency | Handler routing logic |
| Foresight | Hardcoded priority parsing |

### The Translation Loss

The *words* survived but the *meaning* got flattened:

- **"Spatial"** originally meant "awareness of position in workspace/project landscape"
- **"Spatial"** became "EMBEDDED/GRANULAR/DEFAULT response length"

- **"Identity"** originally meant "embodied AI self-awareness"
- **"Identity"** became "return capability list string"

- **"Predictive"** originally meant "pattern recognition and foresight"
- **"Predictive"** became "read priorities from config file"

### Why This Matters for CXO Work

This is the **exact gap** between Level 1 (vision) and Level 3 (implementation) that we identified.

**The opportunity**: Recover the original consciousness model and use it to:
1. Guide how Piper "wakes up" and orients (the litany)
2. Inform the experience quality of each interaction type
3. Connect the philosophical model to the interaction grammar
4. Make "liberal in accepting, strict in emitting" mean something

**The morning standup ritual is the proof case**: It's the only place where the original vision survives—a sequence of orientation queries that helps Piper (and user) get ready for the day. That's what ALL interactions could feel like.
