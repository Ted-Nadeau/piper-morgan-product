## 5 Critical Discoveries

### 1. Navigation Crisis (Score: 700) ⚠️ BLOCKS ALL IMPROVEMENTS

**Problem**: No global navigation menu, no feature discovery mechanism, users must memorize URLs

**Evidence**:
- Journey 1 (Onboarding): User can't find features beyond chat → 😤 Frustrated
- Journey 2 (Daily PM): Must type `/standup` URL manually → 😤 Annoyed
- Journey 5 (Settings): Can't discover configuration options → 😤 Resigned

**Impact**: Blocks all other improvements - highest priority gap

**Solution**: Global navigation menu
- **Effort**: 2-3 days
- **Cost**: $10K
- **Sprint**: 1 (immediate)
- **ROI**: Massive - enables discovery of all features

---

### 2. Document Management Blind Spot (Score: 360 each for G64, G65)

**Problem**: Can upload files and generate artifacts, but **can't browse or retrieve them later**

**User Journey (New Journey 6)**:
1. User uploads PRD template → ✅ Works
2. Piper generates excellent PRD → ✅ Works
3. Next day: "Where's that PRD?" → ❌ **Can't retrieve it**
4. No file browser, no artifact persistence → Lost work

**Impact**: **Core product value proposition undermined**

**Solution**: File browser + artifact persistence
- **Effort**: 1 week
- **Cost**: $15K
- **Sprint**: 5.5 (added to roadmap)
- **Strategic Decision**: Option A (Lightweight) - generic artifact types, defer canonical PM docs to Beta

---

### 3. Theme Inconsistency (Score: 360)

**Problem**: Two completely separate color systems with no unified design system

**Evidence**:
- **Light theme** (home.html, standup.html): `#3498db` blues, hard-coded
- **Dark theme** (learning-dashboard.html, personality.html): `#007acc` blues (VS Code style), hard-coded
- No CSS variables, no design tokens, no governance
- Feels like **different products** when switching pages

**Impact**: Jarring user experience, undermines cohesion

**Solution**: Implement Phase 3 design system
- **Effort**: 2 weeks
- **Cost**: $30K
- **Sprint**: 3-4
- **Deliverable**: **Already created!** tokens.css, light.css, dark.css with 100+ tokens ready to use

---

### 4. Cross-Channel Fragmentation (Score: 140 - architectural)

**Problem**: Web, CLI, and Slack feel like **three separate assistants** with no memory sync

**Evidence**:
- User asks question in Slack → Piper answers
- User switches to web chat → Slack conversation **not visible**
- Settings changed in web → **Unclear if applies to Slack/CLI**
- Mental model breaks: "Is this the same Piper?"

**Impact**: Product differentiation opportunity missed

**Solution**: Unified conversation store
- **Effort**: 4 weeks
- **Cost**: $50K
- **Phase**: Beta Q1 (after MVP)
- **Strategic**: This is the long-term competitive differentiator

---

### 5. Accessibility Gap (Score: 480 for ARIA, 420 for contrast)

**Problem**: No ARIA labels, minimal keyboard navigation, untested color contrast

**Evidence**:
- 0% of interactive elements have ARIA labels
- Can't navigate with keyboard alone (Tab doesn't work everywhere)
- Color contrast unvalidated (likely WCAG failures)
- No screen reader testing

**Impact**: **Legal compliance risk** + excludes users with disabilities

**Solution**: Accessibility pass
- **Effort**: 1 week
- **Cost**: $25K
- **Sprint**: 7
- **Added to CLAUDE.md**: Comprehensive WCAG 2.2 AA guidance for all future work

**Token Categories**:

2. Semantic Colors (primary, success, error, warning, info)
3. Typography (font families, sizes, weights, line heights)
4. Spacing (4px base scale: 4, 8, 12, 16, 24, 32, 48, 64, 96)
5. Border Radius (rounded corners: 4px, 8px, 12px, 16px, full)
6. Shadows (elevation system: 1-5)
7. Animation (duration, easing functions)
8. Z-Index (layering system)

**Implementation Strategy**: Incremental adoption
- Week 1: Add token files, no visual changes
- Week 2-3: Migrate page-by-page (home → standup → learning → personality)
- Week 4: Test theme toggle, verify WCAG AA compliance

**WCAG Compliance**: All color tokens validated for contrast ratios
- Text on background: 4.5:1 minimum
- Large text: 3:1 minimum
- Interactive elements: 3:1 minimum
