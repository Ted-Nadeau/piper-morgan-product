## 12:10 PM - Ted's Agent Architecture Proposal

### Core Concept: Micro-Format Processing Pipeline

**Graph Notation HeteroData**
- Nodes: AI Agents with various types, configurations, capabilities, restrictions
- Flow: Ted → Agent Assistant → Micro-format extraction → Processing cascade

### Micro-Format Types Identified
1. Capability / Feature / Use Case
2. Initiative / Epic / Story / Task
3. Rule / Requirement / Guideline / Heuristic / Algorithm
4. Assertion/Statement (→ research, validate)
5. Question (& draft answer)
6. Open Issue / Change Request / Trouble Report
7. User Permission / Security
8. Data model / Schema
9. Events / Workflow (workitem, workstep)
10. Functions / Objects
11. (Anticipates ~2x more types)

### Relationship Types
Micro-formats relate with: related, blocks, enables, is-dependent-on, etc.

### Processing Architecture
```
Input Layer: Text analysis → micro-format extraction
  ↓
Routing Layer: One listener per micro-format type
  ↓
Processing Layer: ON EVENT new-X DO [predefined skills]
  ↓
Service Layer: Github, Code, Wiki, Text files, Graph-DB, Relational-DB
```

### Key Architectural Principles
- Many small agents (helps with context, security, division of labor, scaling)
- Write-flow vs read-and-work-update flow distinction
- Inter-agent communication evolution: file-based → DB/repository → message → workflow
