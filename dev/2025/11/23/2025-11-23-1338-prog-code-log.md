## Architecture Notes

**Pattern Detection Strategy**:
- Regex patterns anchor to sentence boundaries (`^...$` implicit)
- Resource type detection: Look for "lists?", "todos?", "projects?" keywords
- Extract name by removing resource type from captured text
- Handles variations: "my X", "X", "access to X", etc.

**Error Handling**:
- Missing resource: Returns friendly error message
- Permission denied: Returns permission error
- API failures: Caught and formatted as error response
- Non-matching commands: Fall through to conversational AI

**Reuse of Option B**:
- Uses `openShareModal()` directly - same modal, same API calls
- Uses `canShare()`, `getUserRole()`, `formatRole()` from permissions.js
- Uses `appendMessage()` for chat display
- Uses existing toast notification system
