## 📋 Template notes

### Writing guidelines

**Voice & tone**:
- Professional but conversational
- Technical precision without jargon overload
- Celebrate wins without hyperbole
- Acknowledge challenges without drama
- "We" for team, "I" when appropriate for PM decisions

**Section priorities**:
1. Shipped (most important - what actually got done)
2. Learning pattern (unique value - what we learned)
3. Coming up (sets expectations)
4. Weekend reading (builds community knowledge)
5. Blockers/resources (operational transparency)

**Sentence case for headings**:
- ✅ "Shipped this week" 
- ✅ "Coming up next week"
- ✅ "This week's learning pattern"
- ❌ "Shipped This Week" (Title Case)
- ❌ "SHIPPED THIS WEEK" (All Caps)

**Length targets**:
- Shipped section: 40-50% of content
- Learning pattern: 15-20% of content
- Everything else: 30-40% combined
- Total: 800-1200 words (readable in 5-7 minutes)

### Learning pattern structure

The learning pattern section is what makes Weekly Ships valuable beyond status updates. It captures **transferable insights** that can help other teams/projects.

**Good learning patterns**:
- Specific and concrete (not abstract principles)
- Include real examples from the week
- Show cause and effect clearly
- Applicable beyond immediate context
- Connect to broader methodology

**Example pattern types**:
- Process improvements that compound
- Architecture decisions with rationale
- Coordination patterns between agents/humans
- Debugging approaches that generalize
- Tools/techniques worth spreading

**Bad learning patterns**:
- Generic advice ("tests are important")
- Obvious statements without insight
- Context-specific without generalization
- Vague principles without examples

### Weekly preparation process

**Thursday afternoon** (with team):
- Review session logs from Monday-Thursday
- Identify key achievements and challenges
- Note interesting patterns or decisions
- Draft "Shipped" section bullets

**Friday morning** (PM writing time):
- Complete "Shipped" section
- Write learning pattern (most important!)
- Draft "Coming up next week"
- Add blockers/resources
- Write weekend reading
- Edit for flow and voice

**Friday afternoon** (distribution):
- Post to internal channels (Slack, Notion)
- Cross-post to LinkedIn newsletter
- Archive in project knowledge base
- Track engagement metrics

### Adaptation for different phases

**Alpha/Beta phase**:
- Emphasize user feedback and iteration
- Include tester engagement metrics
- Highlight UX improvements
- Document pain points discovered

**MVP/Launch phase**:
- Focus on feature completion
- Emphasize stability and polish
- Include performance metrics
- Document production readiness

**Post-launch/Growth phase**:
- Highlight adoption metrics
- Share user success stories
- Document scaling challenges
- Emphasize maintenance and support

Update footer links and metrics to match current state:

- Any special announcements
