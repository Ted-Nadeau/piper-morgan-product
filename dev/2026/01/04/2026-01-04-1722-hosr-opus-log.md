## Role Cadence Deep Dives

### 1. Chief Architect

**Session history** (from agent log):
- Dec 1, Dec 2 (early), Dec 16, Dec 25-27 cluster
- Pattern: Strategic intervention points, not daily ops

**Role Evolution** (from xian):

*Origin*: First role defined. Emerged from proof-of-concept phase when xian (PM/UX background) needed architectural partnership to figure out how to actually build Piper.

*Early state*: CA wrote code, xian pasted it in, debugging loop was bad.

*Key evolution*: Separation of concerns emerged:
- **Chief Architect** (Opus): Strategy, planning, oversight. No code.
- **Lead Developer** (Sonnet initially): Oversees implementation, reports to CA
- **Coding agents** (Cursor, then Claude Code): Actual implementation

*Why this separation*:
- Context preservation improved at each level
- Avoided fabrication risk (Lead Dev doesn't write code directly)
- Cleaner handoffs between strategic and tactical work

*Current cadence*: Not daily. Sessions triggered by:
- Advice needed on architectural decisions
- Work completed (review/sign-off)
- Roadmap planning

*Model*: Opus (strategic, complex reasoning)

**Key Insight**: Role separation emerged from solving real problems (bad debugging loops, fabrication risk, context loss)—not designed top-down.
