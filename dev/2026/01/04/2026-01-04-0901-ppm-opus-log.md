### CXO Report Analysis

**Core insight**: "The problem isn't capability—it's discoverability."

Pattern-045 manifests: 35 working canonical queries, but users can't find them.

**Design decisions already made**:
1. **Proactivity**: Assistant mode with trust-gradient modulation (Stage 1→4 progression)
2. **Context persistence**: Three-layer model (conversational memory, user history, composted learning)
3. **Suggestion frequency**: Context-dependent with throttling (max 2 per 5 interactions)

**Discovery patterns recommended**:
- Contextual command palette (triggered by `/` or `?`)
- Proactive capability hints (after successful tasks)
- Empty state education (every empty state teaches)
- "Did you mean" recovery (failed interactions become discovery moments)

**Voice guidelines**: "Professional colleague, not friend" — the Contractor Test.

**FTUX connection**: Discovery failure is partially onboarding failure. FTUX should teach: what Piper is (✓ via PDR-001), how to ask (✗ weak), where to start (✓ okay).

**B2 success criteria defined**:
- Users can discover capabilities without documentation
- "What can you help with?" returns contextual response
- Dead-end interactions offer recovery paths
- Session continuity feels natural
- Proactive suggestions feel helpful, not intrusive
