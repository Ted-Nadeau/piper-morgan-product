### 1. Auth Middleware Fix (#528)
**Problem**: Settings OAuth Connect buttons failed with "Expired token rejected" because `/api/v1/settings/integrations/{provider}/connect` endpoints required JWT auth.

**Root Cause**: OAuth connect endpoints redirect to external provider - no user context needed at that point. State token provides CSRF protection on callback.

**Fix Applied** ([auth_middleware.py:71-75](services/auth/auth_middleware.py#L71-L75)):
```python
# Issue #528: Settings integrations OAuth endpoints
"/api/v1/settings/integrations/slack/connect",
"/api/v1/settings/integrations/slack/callback",
"/api/v1/settings/integrations/calendar/connect",
"/api/v1/settings/integrations/calendar/callback",
```

**Design Decision**: Only `connect` and `callback` are auth-exempt. `disconnect` remains protected (user must be logged in to disconnect).
