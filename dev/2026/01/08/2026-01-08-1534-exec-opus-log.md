## Ideas Discussion

### Idea 1: Claude Cognitive Memory System

**Source**:
- GitHub: https://github.com/GMaN1911/claude-cognitive
- HN discussion: https://news.ycombinator.com/item?id=46438814
- Reddit thread on language-specific guidance

**What it is**:
- Working memory system for Claude Code
- **Context Router**: Attention-based file injection with cognitive dynamics
  - HOT (>0.8): Full file injection
  - WARM (0.25-0.8): Headers only
  - COLD (<0.25): Evicted
  - Files decay when not mentioned, activate on keywords, co-activate with related files
- **Pool Coordinator**: Multi-instance state sharing for long-running sessions
  - Auto-detects completions/blockers
  - Works across 8+ concurrent instances
- Claims 64-95% token savings

**Initial Assessment**:

| Aspect | Relevance to PM |
|--------|-----------------|
| Attention decay model | Interesting parallel to our Moments/attention concepts |
| Multi-instance coordination | Directly relevant to our multi-agent challenges |
| Pool state sharing | Could address subagent logging gap we identified |
| Token optimization | Less critical for us (Anthropic relationship) but still valuable |
| History tracking | Similar to our omnibus log function |

**Questions for deeper analysis**:
1. How does this compare to our existing briefing document strategy?
2. Does the HOT/WARM/COLD model offer something our progressive loading doesn't?
3. Could Pool Coordinator pattern improve our Lead Dev → subagent coordination?
4. Is this complementary or competing with our spatial intelligence MCP approach?

**Recommended action**: CIO-led comparative analysis against our existing patterns
