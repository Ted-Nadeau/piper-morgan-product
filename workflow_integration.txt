=== TECHNICAL INTEGRATION POINTS ===

1. PERSONALITYPROFILE LOADING:
   Database query + PIPER.user.md override mechanism

   Technical Flow:
   - PersonalityProfileRepository.get_by_user_id() queries database
   - ProfileCache.get() checks LRU cache first (<5ms hit, <50ms miss)
   - PIPER.user.md YAML parsing with validation and override application
   - Cache merged profile for subsequent requests
   - Circuit breaker protection if profile loading >50ms

   Integration Points:
   - services/personality/repository.py: Database + YAML integration
   - services/personality/cache.py: LRU caching with TTL management
   - services/configuration/piper_config_loader.py: YAML parsing extension
   - main.py: Profile repository injection into ResponsePersonalityEnhancer

2. ENHANCEMENT PROCESSING:
   Template interception and transformation pipeline

   Technical Flow:
   - TemplateRenderer.render_template() generates initial response
   - PersonalityEnhancer.enhance_response() intercepts template output
   - TransformationService applies: warmth → confidence → actions
   - ActionHumanizer.humanize() applies final formatting
   - Circuit breaker monitors total enhancement time (<100ms)

   Integration Points:
   - services/ui_messages/templates.py: Template interception point
   - services/personality/response_enhancer.py: Core enhancement orchestration
   - services/personality/transformations.py: Content transformation logic
   - services/ui_messages/action_humanizer.py: Final formatting pipeline

3. CONFIDENCE THRESHOLD ADAPTATION:
   Intent confidence drives personality adaptation

   Technical Flow:
   - Intent confidence (0.0-1.0) from intent classification
   - ResponseContext creation with confidence + intent details
   - PersonalityProfile.adjust_for_context() adapts profile
   - Transformation rules based on confidence thresholds:
     * <0.3: High warmth, hidden confidence, more guidance
     * 0.3-0.7: Medium warmth, descriptive confidence
     * >0.7: Professional tone, contextual confidence
     * >0.8: Technical confidence display

   Integration Points:
   - services/intent_service/classifier.py: Confidence score source
   - services/personality/personality_profile.py: Context adaptation logic
   - services/personality/transformations.py: Threshold-based rules

4. PERFORMANCE CONSTRAINT ENFORCEMENT:
   <100ms total enhancement with monitoring

   Technical Flow:
   - Enhancement timer starts at template interception
   - Profile loading timeout: 50ms
   - Transformation timeout: 40ms
   - Monitoring overhead: 10ms
   - Circuit breaker opens if >100ms or >10 failures/minute
   - Graceful degradation returns original response

   Integration Points:
   - services/personality/response_enhancer.py: Master timeout enforcement
   - services/personality/cache.py: Cache performance monitoring
   - services/observability/: Enhancement metrics collection
   - services/api/errors.py: Graceful degradation handling


5. USER EXPERIENCE FLOW:
   How personality preferences affect response display

   User Experience Flow:
   - User configures personality via PIPER.user.md or web interface
   - PersonalityProfile preferences load with validated overrides
   - Enhancement applies user's warmth_level (0.0-1.0) to response tone
   - Confidence display adapts to user's confidence_style preference:
     * numeric: "87% confident"
     * descriptive: "high confidence"
     * contextual: "Based on recent patterns..."
     * hidden: No confidence shown
   - Action orientation drives next-steps generation:
     * high: Every response has explicit actions
     * medium: Actionable when relevant
     * low: Minimal action orientation

   UI Integration Points:
   - PIPER.user.md: Direct personality configuration
   - Web interface: Personality preference forms
   - Response display: Enhanced content with personality applied
   - Error messages: Personality-consistent error handling

6. CONFIDENCE INDICATOR PLACEMENT:
   Context-aware confidence display in responses

   Display Strategy:
   - Intent confidence determines base confidence level
   - User confidence_style preference controls display format
   - Confidence placement varies by response type:
     * CLI: Text-based confidence indicators
     * Web: Visual confidence meters/badges
     * Slack: Contextual confidence language
   - Low confidence triggers supportive language regardless of style

   Implementation:
   - TransformationService.inject_confidence() applies user preference
   - Response templates include confidence placeholder variables
   - UI renders confidence indicators per interface requirements

7. CROSS-INTERFACE CONSISTENCY:
   Unified personality across CLI, Web, Slack

   Consistency Strategy:
   - Same PersonalityProfile applies to all interfaces
   - Interface-specific adaptation while maintaining core personality
   - Standup responses match chat personality via StandupToChatBridge
   - Error handling maintains warmth across all interfaces

   Interface Adaptations:
   - CLI: Text-based warmth and confidence
   - Web: Visual enhancements with personality-driven UI elements
   - Slack: Conversational personality with appropriate emoji/formatting
   - Standup: Personality-enhanced metrics presentation
