=== SERVICES/PERSONALITY/ MODULE STRUCTURE ===

services/personality/
├── __init__.py                  # Module exports and initialization
├── response_enhancer.py         # ResponsePersonalityEnhancer aggregate root
├── personality_profile.py       # PersonalityProfile entity + value objects
├── transformations.py           # TransformationService domain service
├── standup_bridge.py           # StandupToChatBridge domain service
├── repository.py               # PersonalityProfileRepository interface + impl
├── cache.py                    # Profile caching implementation
├── models.py                   # Database models (SQLAlchemy)
└── exceptions.py               # Domain-specific exceptions

Key Implementation Details:

1. ResponsePersonalityEnhancer (response_enhancer.py):
   - Aggregate root following DDD model
   - enhance_response() core domain operation
   - Orchestrates profile loading, transformation, metrics
   - Integration with TemplateRenderer pipeline
   - Circuit breaker for performance protection

2. PersonalityProfile (personality_profile.py):
   - Domain entity with business logic
   - adjust_for_context() method for situational adaptation
   - Value objects: ConfidenceDisplayStyle, ActionLevel, etc.
   - PIPER.user.md override logic
   - Validation rules for personality parameters

3. TransformationService (transformations.py):
   - Domain service for content transformation
   - add_warmth(), inject_confidence(), extract_actions()
   - Rule-based transformations (no LLM calls)
   - Preservation of technical accuracy
   - Transformation order: warmth → confidence → actions

4. StandupToChatBridge (standup_bridge.py):
   - Domain service for standup integration
   - adapt_standup_for_chat() conversion logic
   - apply_personality_to_standup() enhancement
   - Data integrity preservation (metrics, links)
   - Integration with existing standup_formatting.py

5. PersonalityProfileRepository (repository.py):
   - Repository pattern implementation
   - get_by_user_id(), save(), get_default() methods
   - Database + PIPER.user.md integration
   - Cache integration for performance
   - SQLAlchemy async session management

6. Profile Caching (cache.py):
   - LRU cache with thread safety
   - Cache warming and invalidation strategies
   - Performance monitoring and metrics
   - Circuit breaker integration
   - TTL management and background refresh

7. Database Models (models.py):
   - SQLAlchemy model for personality_profiles table
   - Enum definitions for confidence_style, action_orientation
   - Migration-friendly schema design
   - Relationship with existing user models
   - Index strategies for performance

8. Exception Handling (exceptions.py):
   - PersonalityEnhancementError base exception
   - ProfileLoadError, TransformationError subclasses
   - Circuit breaker exceptions
   - Performance timeout exceptions
   - Graceful degradation error handling

Module Dependencies:
- services.database (models, session management)
- services.configuration (PIPER.user.md integration)
- services.ui_messages (ActionHumanizer, TemplateRenderer)
- services.shared_types (existing enums and types)
- services.utils (caching, performance monitoring)

Performance Characteristics:
- <100ms total enhancement time
- <5ms cache hit profile retrieval
- <50ms cache miss profile loading
- Circuit breaker protection for reliability
- Graceful degradation on errors/timeouts
