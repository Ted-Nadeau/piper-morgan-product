=== INTEGRATION ARCHITECTURE ===

1. TEMPLATE LAYER INTEGRATION:
   PersonalityEnhancer → ActionHumanizer pipeline integration

   Current Flow (Extended Phase -1 Discovery):
   FastAPI → Intent → Template → ActionHumanizer → Output

   New Flow (Personality Enhanced):
   FastAPI → Intent → Template → PersonalityEnhancer → ActionHumanizer → Output

   Integration Point (services/ui_messages/templates.py):
   - TemplateRenderer.render_template() method
   - Insert PersonalityEnhancer after template formatting
   - Before ActionHumanizer.humanize() call

   Modified Pipeline:
   1. TemplateRenderer renders template with context
   2. PersonalityEnhancer.enhance_response() processes output
   3. ActionHumanizer.humanize() applies final formatting
   4. Return enhanced response to caller

2. CONFIDENCE INTEGRATION:
   Using discovered thresholds (0.3, 0.7, 0.8) for personality adaptation

   Confidence Thresholds from classifier.py Discovery:
   - Low: <0.3 (needs clarification) → High warmth, hidden confidence
   - Medium: 0.3-0.7 (learning opportunity) → Medium warmth, descriptive confidence
   - High: >0.7 (confident) → Professional tone, contextual confidence
   - Very High: >0.8 (proceed automatically) → Technical confidence display

   Adaptation Rules:
   - Warmth inversely correlated with confidence (high conf = less warm)
   - Confidence display style matches user preference + intent confidence
   - Action orientation increases with lower confidence (more guidance needed)
   - Technical depth adapts to user preference + confidence level

3. STANDUP BRIDGE ARCHITECTURE:
   StandupToChatBridge implementation approach

   Current Standup Flow (Phase -1 Discovery):
   web/app.py:475-481 → format_standup_metrics() → JSON response

   New Standup Bridge Flow:
   Standup Service → StandupToChatBridge → PersonalityEnhancer → Chat Format

   Bridge Responsibilities:
   - Convert standup metrics to conversational format
   - Apply personality to standup-specific content
   - Maintain data integrity (metrics, links, issue numbers)
   - Unify experience with main chat responses

   Implementation Strategy:
   - StandupToChatBridge as domain service
   - Integration at standup_formatting.py level
   - Personality applied to human-readable format
   - Bridge output feeds standard response pipeline
